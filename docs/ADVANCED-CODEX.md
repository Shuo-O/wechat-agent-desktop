# Codex 高级模式

## 适用场景

- 你希望让微信里的助手能回答当前项目代码相关问题
- 你希望让助手基于本地仓库给出修改建议
- 你接受它在高级模式下读取，甚至修改指定工作目录里的文件

## 前置条件

在使用前，先在终端确认：

```bash
which codex
codex login
```

如果 `codex` 没装或没登录，应用里的 Codex 模式会失败。

## 配置步骤

1. 打开“显示高级助手”
2. 在助手模式里选择 `Codex（高级）`
3. 选择一个工作目录
4. 选择权限模式：
   - `只读问答`
   - `允许在工作目录内修改文件`
5. 保存设置

## 权限模式说明

### 只读问答
- 推荐默认使用
- 适合：
  - 问代码结构
  - 问报错原因
  - 问应该怎么改
- 不应该真正修改文件

### 允许在工作目录内修改文件
- 面向更高级的使用者
- 适合：
  - 让 Codex 直接做代码修改
  - 让 Codex 执行需要写文件的操作
- 使用时要清楚：
  - 它可能会改工作目录里的文件
  - 当前应用只会把 Codex 的最终文字回复发回微信

## 当前实现方式

应用内部会调用：

```bash
codex -a never -s <sandbox> -C <workdir> exec --skip-git-repo-check -o <tempfile> "<prompt>"
```

这意味着：
- 工作目录就是 Codex 观察和操作的根目录
- 最终回复会被收集后回发到微信

## 当前限制

- 还没有在 UI 里展示 Codex 的 diff 或文件清单
- 还没有自动帮用户安装 `codex`
- 还没有自动处理 `codex login`

## 建议

- 第一次使用先选“只读问答”
- 先拿一个小仓库做测试
- 真要让它改文件时，再切到可写模式
