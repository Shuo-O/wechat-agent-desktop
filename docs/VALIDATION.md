# 手工验证指南

## 验证目标

当前版本以手工验证为主，建议按三条路径做：

1. 微信链路验证
2. DeepSeek 真实回复验证
3. Codex 高级模式验证

## 前置条件

- 本机可以正常启动 Electron 应用
- 有可扫码登录的微信环境
- 如果验证 DeepSeek，需要一个可用的 DeepSeek API Key
- 如果验证 Codex，需要本机已经：
  - 安装 `codex`
  - 执行过 `codex login`

## 启动应用

```bash
npm install
npm run start
```

## 路径一：微信链路验证

目的：
- 验证二维码显示、扫码登录、长轮询和文本回发是否正常

步骤：
1. 启动应用
2. 保持助手模式为 `演示助手`
3. 点击“开始扫码登录”
4. 用微信扫码并在手机上确认
5. 点击“开始接收消息”
6. 用微信给 Bot 发一条私聊消息

通过标准：
- 界面状态变为“微信已登录”
- 运行状态变为“运行中”
- 联系人列表出现对应联系人
- 微信里收到一条演示助手回复

## 路径二：DeepSeek 真实回复验证

目的：
- 验证真实模型接入是否正常

步骤：
1. 在“助手设置”里选择 `DeepSeek`
2. 模型先用 `deepseek-chat`
3. 填入 `DeepSeek API Key`
4. 建议勾选“保存时清空历史上下文”
5. 点击“保存设置”
6. 用微信给 Bot 发送一条新私聊消息

建议测试语句：

```text
请直接用中文介绍一下你自己，不要说自己是演示助手。
```

通过标准：
- 回复不再是固定模板
- 回复内容明显来自真实模型
- 日志里没有接口报错

如果失败，先看：
- 是否忘了保存设置
- 是否沿用了旧联系人历史
- API Key 是否可用

## 路径三：Codex 高级模式验证

目的：
- 验证本地代码助手接入是否正常

步骤：
1. 打开“显示高级助手”
2. 助手模式选择 `Codex（高级）`
3. 选择一个本地工作目录
4. 权限模式先选 `只读问答`
5. 建议勾选“保存时清空历史上下文”
6. 点击“保存设置”
7. 用微信发一条与该目录相关的代码问题

建议测试语句：

```text
请用中文告诉我这个项目是做什么的，主要入口文件在哪里。
```

通过标准：
- 回复内容与所选工作目录有关
- 回复中不再出现演示助手文案
- 日志中没有 `Codex 执行失败`

## 常见问题

### 二维码不显示

- 当前实现会把登录链接转成本地二维码图片
- 如果仍不显示，优先看运行日志是否有登录接口异常

### DeepSeek 看起来像演示助手

- 通常不是接口没生效，而是旧历史上下文没清掉
- 重新保存设置时勾选“保存时清空历史上下文”

### Codex 模式没有反应

优先在终端确认：

```bash
which codex
codex login
```

如果命令不存在或未登录，应用里的 Codex 模式会失败。

## 当前验证边界

- 这套验证主要覆盖私聊文本链路
- 还不覆盖群聊、图片、文件、语音
- 还没有自动化回归测试，修改后建议至少重跑三条手工路径
