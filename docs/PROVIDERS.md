# Provider 说明

## 当前支持的助手类型

## 1. 演示助手

适合：
- 验证微信登录和消息闭环
- 不接真实模型时快速测试

特点：
- 不需要 API Key
- 不会调用外部模型
- 回复内容固定偏演示性质

## 2. DeepSeek

适合：
- 普通用户默认使用
- 需要真实聊天和问答能力

特点：
- 应用内置官方地址
- 只需要填写 `DeepSeek API Key`
- 默认模型是 `deepseek-chat`
- 可选 `deepseek-reasoner`

## 3. OpenAI 兼容

适合：
- 想接其它兼容 `/chat/completions` 的模型服务

特点：
- 用户自己填写 Base URL、Model、API Key
- 适合更灵活的兼容型接入

## 4. Codex（高级）

适合：
- 代码仓库问答
- 本地项目分析
- 在工作目录里让 Agent 给出修改建议，或直接执行改动

特点：
- 依赖本机 `codex` 命令
- 依赖本机已有 `codex login`
- 支持选择工作目录
- 支持两种权限模式：
  - `只读问答`
  - `允许在工作目录内修改文件`

## Provider 选择建议

### 面向普通用户
- 默认优先 `DeepSeek`

### 面向想接其它模型的人
- 用 `OpenAI 兼容`

### 面向做代码问题分析的人
- 开启高级模式后使用 `Codex`

## 为什么不把 Codex 作为默认入口

- 它依赖本地安装和登录
- 它涉及工作目录和文件权限
- 对普通用户来说学习成本更高

所以产品策略是：
- 普通用户先看见 DeepSeek
- 更懂技术的人再打开高级模式使用 Codex
